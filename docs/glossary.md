# APOLLO Glossary

## Terms

| Term | Meaning in APOLLO |
| --- | --- |
| `Tracer` | One narrow implementation slice that widens the repo only as far as the current proof requires |
| `ATHENA` | The upstream physical-truth service that publishes identified arrival and departure events |
| `claimed_tags` | The mapping from an ATHENA identity hash to a specific APOLLO member record |
| `lobby eligibility` | A derived read that says whether a member can appear in the open lobby based on explicit profile state |
| `ARES` | The later matchmaking and rating subsystem planned for APOLLO |
| `deterministic recommendation` | A recommendation derived from explicit workout history without generated planning or LLM behavior |
| `member intent` | The state APOLLO owns: profile preferences, visibility, availability, workouts, and later matchmaking intent |
